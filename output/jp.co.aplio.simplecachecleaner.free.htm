#Summary

Simple cache Cleaner is an application that can easily erase cache. Erase cache and make the operation of an Android device comfortable ! An easy way to handle it ! Activate the application, then just push a button of "Clean".

#Functions
- Bulk deletion of cache.
- Sorting
- Uninstalls the application
- Displays a cache size of each application
- Transition to the list of services activated
- Simultaneous termination with "Cleen button".

#Features

1. Non-resident software-Andriod2.3 or more
Most of popular cache-cleaners are resident type, therefore they continue to be being activated and be consuming memory. But "Simple Cache Cleaner" is a non-resident type.
This is designed to quit itself quickly after having terminated an application by "HOME button" as well as "RETURN button".
You would see the big difference, if compared to other applications.

"Simple Cache Cleaner" does not implement such contradictory function that "cache cleaning" itself continues to be a resident, and consume memory.

Mainly, the following function would never be implemented now and in the future.

- One-tap-cleaning by widget
- Automatic-deleting or informing, by supervising memory consumption in background.

2. Inter working with "Task Killer"
Inter working function with the simple and high-peformance task killer, "Simple Task Killer", is equipped. By this function, only one button operation can delete cache and terminate a task, it is very recommended to try this inter working function.
*: In order to enable the inter working function, you would be required to install this application, as well as "Simple Task Killer".

#Q&A

Q: Can this application save power ?
A: This application is designed for the purpose of intentional releasing of memory, which has been occupied by other application. It is not for enhancing the effeciency of saving power.

Q: What is cache ? Is it better to delete ?
A: Cache is the area for an application to temporarily stores its frequent use data.

A lot of Android devices do not have abundant amount of memory. Therefore, the operation of an Android device could become comfortable by regularly deleting cache.

Q: Doesn't this implement an widget ?
A: The widget is always a resident, so "Simple Cache Cleaner" would not implement it now and in the future.

Q: Doesn't this delete memory automatically ?
A: Like the widget, that function is always a resident, so "Simple Cache Cleaner" does not implement it now, and also wouldn't have any plan to implement a resident function in the future.

Q: A short-cut has disappeared, why ?
A: When being moved into SD card, a short-cut may disappear. This seems to occur in some kind of devices by malfunction of itself. In such a case, it is recommended not to move this application to SD card.

Q: It has disappeared from the notification-bar , why ?
A: This has occured by the specification of Android OS. When this has once been moved into SD memory, that application will not become active again even after the device would have been re-booted.

Q: The cache does not seem to have been deleted, why ?
A: This problem may be caused by distinct malfunction of some kind of Andriod devices. If this problem could not been resolved even after re-installing this application, please avoid to use it.

#Q&A

Q. Want to kill application by one-toutch. Doesn't STK implement an widget ?

A. An widget is always a resident, so STK does not implement it. Paid STK is equipped with a one-toutch killing function. (Android4.0 or more)

Q. Even after terminating a application, however it immediately appears again. Why ?

A. This is caused by the specification of Android OS. The pre-installed application which belongs to the configuration type, that needs to be continuing activated such as Facebook, is structured to reboot automatically .
Any function can terminate operation of an application temporarily, but no function would be able to re-terminate such application that reboots itself even after terminated once.

Q. Want STK that will not re-boot an application.

A. STK's basic concept is not to implement a resident function, therefore STK has no plan to implement a supervisory function that requires an application to suppress self-reboot.

Q. How to use the ignore-list ?

A. Those applications registered in the ignore-list will disappear from the application list, which appears after STK is activated.
This means that STK ignores whether they are operating or not. In other words they have already been put outside of STK's management.
Such applications that need to be always activated as "An alarm clock" and "E-mail", should not to be registered in the ignore-list to avoid a risk of terminating their operation.
When add an application in the ignore-list, first tap its name in the application pane, then next pane will pop up, finally tap "Add to ignore list".

常見問題：
1.什麼是快取？
答：快取是一種將數據存儲在高速內存中，加速應用程式讀取速度，提升應用程式性能的機制。

2.為什麼要清理快取？
在android手機中，快取文件存儲在手機內部記憶體中（ROM），手機內部記憶體不同於SD卡，可以理解為電腦中的內存和硬盤，一個速度快一個速度慢。
因為手機內部記憶體的容量是有限的，如果快取過多，而沒有及時清理，則會導致內部記憶體不足，快取機制失效而降低應用程式的性能，並且不能安裝新的應用程式。

3.清理後會不會丟失應用程式的數據？
答：快取跟應用程式的數據不同，快取是臨時存儲的文件，使用後就失效了；而數據是應用程式需要使用的記錄，如游戲記錄，用戶信息等。
清理快取不會清理應用程式的數據，請不用擔心

#Q&A

Q. Want to kill application by one-toutch. Doesn't STK implement an widget ?

A. An widget is always a resident, so STK does not implement it. Paid STK is equipped with a one-toutch killing function. (Android4.0 or more)

Q. Even after terminating a application, however it immediately appears again. Why ?

A. This is caused by the specification of Android OS. The pre-installed application which belongs to the configuration type, that needs to be continuing activated such as Facebook, is structured to reboot automatically .
Any function can terminate operation of an application temporarily, but no function would be able to re-terminate such application that reboots itself even after terminated once.

Q. Want STK that will not re-boot an application.

A. STK's basic concept is not to implement a resident function, therefore STK has no plan to implement a supervisory function that requires an application to suppress self-reboot.

Q. How to use the ignore-list ?

A. Those applications registered in the ignore-list will disappear from the application list, which appears after STK is activated.
This means that STK ignores whether they are operating or not. In other words they have already been put outside of STK's management.
Such applications that need to be always activated as "An alarm clock" and "E-mail", should not to be registered in the ignore-list to avoid a risk of terminating their operation.
When add an application in the ignore-list, first tap its name in the application pane, then next pane will pop up, finally tap "Add to ignore list".

　バッテリーゲージは、どんなアプリケーションよりも前面にバッテリー（電池）の残量と日付を表示する事が出来るアプリです。フルスクリーンになってしまうアプリを立ち上げていて、バッテリーの消費に気付かず電源が落ちてしまった事はありませんか？

そんな問題を解消するのがバッテリーゲージです！

　起動すると設定画面が出てきますので、どこにどのように表示したいかを選択し、最後に左下に表示されている「開始・更新」ボタンを押すとバッテリーの情報が最前面にオーバーレイ表示されます。表示を確認したら、戻るボタンでアプリを終了すれば完了です。

無料版ではバーの色が赤色のみで、文字色・背景色が変更できないなどの制限がありますが、多機能なProバージョンでは文字とゲージの大きさや色の変更、透明度の変更など幅広いカスタムが可能です！

特に、背景色と透明度の変更により無料版と比較すると見た目が劇的に変化しますので、気に入って頂いた方は、ぜひ多機能なProバージョンをご利用下さい。

■自動で停止するタスクキラーにご注意ください
このアプリは常に立ちあがっていないと表示できないため、タスクキラー等で停止されると表示が消えてしまいます。しばらくして表示が消えてしまう方は、タスクキラーの「自動キル」によりバッテリーゲージが停止されてしまった可能性があります。

バッテリーゲージはAndroidの「サービス」として開発しており、自動的に再稼働するように設計されておりますが、消えてしまうのが気になる方はタスクキラーの「無視リスト」へ当アプリを追加して頂くと定期的に消える現象は回避できる可能性が高いです。

もしそれでも定期的に停止されてしまう場合は何らかの不具合が考えられますので、下部に記載されているメールアドレスまで状況の説明をなるべく細かく書いてご連絡頂けますようお願い致します。

The "Web Clip" provides a screen-capturing-function to save the page shown by a browser as an image.

#How to use

1. On Browser, open the page that is intended to be saved.
2. Select "Share" from a menu of Browser.
3. Select "Web Clip" from a list of applications.

* In case an web Browser is Chrome, rarely "Web Clip" doesen't look like to be shown on the list.
In order to show it just slide down the application list.
Activate "Web Clip" to confirm stored images.


#Functions

- Can categorise stored images.
- Moves to a next image by flicking the current image to the right or left.
- A file manager, in cooperation with Gallery, can back up images in a simple way.
- Partially expands a thumbnail of the stored image to see it easily.
- The progressing-bar shows a proceeding status of its completion.
- Can share URL obtained an web site on Facebook or Twitter.


#Notes

The image saved by Web Clip is being stored in Gallery.

Since some device may delete the stored images, or may not display by its malfunction, it is strongly recommended to back up them frequently.

Deletion of the image itself means eliminating it from Web Clip.


Note that those shown below can not be saved.

1. The page which needs some action. For example, the page which cannot be read without clicking "more" as shown on the news site, may not be saved.

2. Those applications other than Browser.

3. A page that needs log in like Facebook.


#Q&A

Q: Can Web Clip change the position of an image ?
A: Web Clip has a link with the standard function of Android, then this can not change the position of an image finely, but can bring close it to your favorite position by repeating "On top".

Q: Can Web Clip work with other applications ?
A: This is exclusive use for WebBrowser.

Q: Can not save a page.
A: Capturing may not be possible depending on the desgin of Web site. This may be caused by the problem of Web site or Browser.

Q: Can restore the erased data ?
A: Any deleted data can never be restored, then again it is strongly recommended to frequently back up your precious data.

Q: Where is the data stored ?
A: It is stored in Pictures folder.

Q: Data can't it be stored in a SD card ?
A: Various devices have various position of a SD card, then Web Clip can not directly save data to a SD card. It can be transfered from the folder used for storing.

Q: Quarity of an image is not good.
A: Depending on an amount of memory some image can not be shown beautiful. When activate Gallery from a menu it may finely be shown.
