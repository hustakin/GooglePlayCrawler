说明
Open-source SSH and telnet client with SCP file transfer. Based on original ConnectBot.

Feature enhancements:
- background file transfer (SCP protocol)
- character picker dialog (on-screen button and hardware SYM key)
- single line input (on-screen button)
- tap and hold menu (switch to fullscreen mode, change font size)
- save SSH keys as files
- save console screenshots (PNG format)
- new key mappings (hardware keyboard)
- ssh-agent (by Roberto Tyley)
- x11 forwarding
- lots of bugfixes

VX ConnectBot provides additional customization for smartphones with a physical keyboard.
Currently, there are customizations for the following smartphones:
- full hardware keyboard (e.g. bluetooth keyboard)
- Samsung Captivate Glide (SGH-i927)
- Sony Ericsson Xperia pro (MK16i)
- Sony Ericsson Xperia mini pro (SK17i)

开放-源代码的SSH和telnet的客户端与SCP的文件传输。基于上的原来的ConnectBot。

功能增强功能：
- 后台文件传输（SCP协议）
- 拾字符器“对话框中（屏幕上的-的的按钮，并硬件SYM键）
- 单线输入（屏幕上的-的按钮）
- 请点选并按住菜单（切换到全屏模式，请更改字体大小）
- 保存作为文件的SSH金钥
- 保存控制台的截图（PNG格式的）
- 新的金钥映射（硬件键盘）
- 的ssh-agent（由Roberto Tyley）
- x11的的的转发
- 大量的的错误修正

VX ConnectBot提供了额外的的的为与一个物理的键盘的智能手机的定制。
目前，有是为的下面的的智能手机的自定义项：
- 全的硬件键盘（例如：蓝牙键盘）
- 三星的Captivate滑翔（SGH-i927）
- 索尼爱立信Xperia的亲（MK16i）
- 索尼爱立信Xperia迷你的亲（SK17i）

更新：由于账户部分逻辑已重新实现，因此在升级到 1.5.X 版本前，请卸载 1.4.X 及之前版本。

如果您在使用中遇到任何问题，请前往项目主页反馈，在市场中评论或留言并不能有效的解决您的问题。

注意事项：

1. 受限于手机性能，在使用本地SOCKS代理情况下，耗电量将大大增加，部分机型上还会遇到崩溃的情况。因此建议不要启用 “使用SOCKS代理”选项，并自行在服务器端部署HTTP(S)代理，或向服务商咨询远程代理的参数。

2. 请确认你的手机已 ROOT。在获得 ROOT 权限的前提下，安全隧道支持 WIFI, GPRS （CMNET）, EDGE 和 3G 等联网方式。并提供了设置全局代理，以及为指定程序设置代理的功能。

欢迎通过 Twitter 联系我们: @sshtunnel_dev

快速设置指南

使用本软件前请确认满足以下两个条件中的任意一个：1. 有自己的 VPS 或者独立服务器；2. 有专门服务商提供的 SSH 帐号。

服务器设置

满足第二项条件的用户可跳过此部分。

1. 在 VPS 或者服务器上安装任意 Linux 操作系统。

2. 安装 Squid 代理服务器。Ubuntu / Debian : sudo apt-get install squid, Redhat / CentOS / Fedora : sudo yum install squid

3. 以默认配置启动 Squid 即可。

手机设置

1. 下载 SSHTunnel 最新版

2. 启动后填入以下参数：

Host 服务器地址
Port 一般为 22，Puff 用户还可填 443
User, Password 你的 SSH 用户名与密码
Local Port 任意大于 1024 的数字即可，如 1984
PUFF 用户和自建服务用户，Remote Port 填入 3128
其他用户请选择“使用SOCKS代理”选项，Remote Port 留空即可

3. 点击 Connect 按钮

4. 非 Root 用户还需要手动设置手机的代理，如 127.0.0.1:1984
